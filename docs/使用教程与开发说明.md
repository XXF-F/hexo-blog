# 狼辛博客 · 使用教程与后台开发说明

> Hexo 6 + Butterfly 主题 + 自研管理后台  
> 线上博客：https://xxf-f.github.io  
> 管理后台：http://207.57.123.17:4001

---

## 一、项目是什么

这是一套「**静态博客 + 可视化管理后台**」方案：

| 部分 | 技术 | 作用 |
|------|------|------|
| 博客前台 | Hexo + Butterfly | 访客看到的文章、归档、标签页 |
| 线上托管 | GitHub Pages | 免费 HTTPS，地址 `用户名.github.io` |
| 管理后台 | Node.js + Express | 写文章、改主题、看留言 |
| 留言服务 | 同上（4001 端口） | 访客留言仅管理员可见 |
| 收藏功能 | 浏览器 localStorage | 访客本地收藏，不上传服务器 |

### 架构图

```
┌─────────────────────────────────────────────────────────┐
│  访客浏览器                                              │
│  https://xxf-f.github.io  （GitHub Pages 静态页面）      │
│       │ 留言提交                                         │
│       ▼                                                 │
│  http://服务器IP:4001/api/comments  （你的服务器 API）   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  管理员浏览器                                            │
│  http://服务器IP:4001  （管理后台）                      │
│       │ 保存文章                                         │
│       ▼                                                 │
│  服务器 /root/hexo-blog/source/_posts/*.md               │
│       │ hexo generate + hexo deploy                     │
│       ▼                                                 │
│  GitHub → xxf-f.github.io 更新                          │
└─────────────────────────────────────────────────────────┘
```

**核心结论：** 后台「保存文章」≠ 前台立刻显示。非草稿文章保存后会自动执行 `hexo generate` + `hexo deploy`，把静态文件推到 GitHub，前台才会更新。

## 二、快速上手（使用者）

### 2.1 登录管理后台

1. 浏览器打开：http://207.57.123.17:4001
2. 默认账号：`admin` / `admin123`（**首次登录后务必在「账号管理」修改密码**）
3. 左侧菜单进入各功能模块

### 2.2 写文章

1. 点击 **写文章** 或 **新建文章**
2. 填写：标题、标签、分类、封面（点「上传封面」选图）
3. 正文支持 Markdown，右侧实时预览
4. 工具栏可插入图片、从媒体库选择
5. **不要勾选「保存为草稿」** → 点 **保存文章**
6. 成功提示「文章已创建并发布到 GitHub」后，等 1～2 分钟刷新前台

| 操作 | 说明 |
|------|------|
| 新建文章 | 侧边栏「写文章」，会清空编辑器 |
| 编辑文章 | 「文章」列表点「编辑」 |
| 草稿 | 勾选「保存为草稿」只存服务器，不部署 |
| 上传封面 | 封面区点「上传封面」，自动进媒体库 |

### 2.3 媒体库

- 路径：**媒体库**
- 支持拖拽上传图片
- 可插入到文章、复制链接、删除文件
- 文件保存在 `source/images/`

### 2.4 主题设置

- 路径：**主题设置 → 可视化设置**
- 常用项：
  - **全站背景图**：上传后点「应用背景」
  - **导航菜单**：可视化添加名称、链接、图标
  - **显示模式**：浅色 / 深色
  - **特效开关**：雪花、点击爱心、烟花等
- 保存后会重新生成并部署（耗时约 30 秒～2 分钟）

### 2.5 留言管理

- 前台文章底部为 **「给我留言」** 表单（非公开评论）
- 访客提交后仅显示「留言已发送」
- 管理员在 **留言** 页面查看：称呼、联系方式、内容、来源页面
- 可删除留言

### 2.6 收藏功能（前台）

- 文章页有「收藏」按钮，首页卡片有 ☆ 快捷收藏
- 导航栏 **我的收藏** 查看列表
- 数据存在访客浏览器本地，换设备会消失

### 2.7 生成部署

路径：**生成部署**

| 按钮 | 命令 | 何时使用 |
|------|------|----------|
| 清理缓存 | `hexo clean` | 生成异常时 |
| 重新生成 | `hexo clean && hexo generate` | 仅本地预览 |
| 部署上线 | `hexo clean && hexo generate && hexo deploy` | 手动同步到 GitHub |

> 正常发文章会自动部署；若前台没更新，手动点「部署上线」。

### 2.8 内容安全

全平台（含管理员）禁止黄赌毒相关内容，以下保存时会被拦截：

- 文章、留言、主题配置、自定义 CSS、媒体文件名等

## 三、部署与运维

### 3.1 服务器要求

- 系统：CentOS 8 / Ubuntu 等 Linux
- 软件：Node.js 20+、Git、PM2
- 开放端口：**4001**（管理后台 + 留言 API）
- DNS：需配置 nameserver（如 `8.8.8.8`），否则无法访问 GitHub

### 3.2 首次部署服务器

```bash
# 克隆项目
git clone https://github.com/XXF-F/hexo-blog.git /root/hexo-blog
cd /root/hexo-blog
npm install
cd admin-panel && npm install && cd ..

# 配置 GitHub 部署 Token（用于自动发布到 Pages）
echo "你的GitHub_Token" > admin-panel/data/deploy-token.txt
chmod 600 admin-panel/data/deploy-token.txt

# 启动后台
npm install -g pm2
pm2 start admin-panel/server.js --name hexo-admin
pm2 save && pm2 startup
```

### 3.3 GitHub Token 获取

1. GitHub → Settings → Developer settings → Personal access tokens
2. Generate new token (classic)，勾选 **repo**
3. 写入服务器 `admin-panel/data/deploy-token.txt`
4. **切勿把 Token 发给他人或提交到 Git**

### 3.4 日常更新流程

```
管理后台改内容 → 保存（自动 deploy）→ 刷新 xxf-f.github.io
```

手动命令（服务器上）：

```bash
cd /root/hexo-blog
git pull                    # 拉取代码更新
pm2 restart hexo-admin      # 重启后台
npx hexo deploy             # 手动发布博客
```

### 3.5 本地开发（可选）

```bash
cd ~/hexo-blog
npm install
cd admin-panel && npm install && cd ..

# 终端 1：管理后台
node admin-panel/server.js    # http://localhost:4001

# 终端 2：博客预览
npx hexo server               # http://localhost:4000
```

## 四、后台程序设计思路

### 4.1 为什么不用 hexo-admin 插件

原项目自带 `hexo-admin` 插件，但我们额外开发了 `admin-panel/`，原因：

| 需求 | hexo-admin | 自研后台 |
|------|-----------|----------|
| 中文可视化菜单编辑 | 弱 | ✅ 表单化 |
| 背景图一键上传应用 | 无 | ✅ |
| 私密留言 | 无 | ✅ |
| 黄赌毒内容过滤 | 无 | ✅ |
| 保存后自动 deploy 到 GitHub | 无 | ✅ |
| 与 Butterfly 主题深度整合 | 一般 | ✅ |

### 4.2 技术选型

```
前端：原生 HTML + CSS + JavaScript（无 React/Vue，轻量易改）
后端：Express.js（单文件 server.js，约 600 行）
存储：文件系统（Markdown、JSON、YAML），无数据库
部署：child_process 调用 hexo 命令行
```

**设计原则：**

1. **文件即数据库** — 文章是 `.md`，配置是 `.yml`，留言是 `.json`，直观、易备份
2. **后台不改 Hexo 核心** — 通过读写 Hexo 标准目录（`source/_posts`、`_config.butterfly.yml`）协作
3. **静态与动态分离** — 前台静态托管 GitHub；动态 API 放服务器
4. **单进程够用** — 个人博客无需微服务，PM2 守护一个 Node 进程即可

### 4.3 目录结构

```
hexo-blog/
├── _config.yml                 # Hexo 主配置（站点名、URL、deploy）
├── _config.butterfly.yml         # Butterfly 主题覆盖配置
├── source/
│   ├── _posts/                 # 文章 Markdown（后台读写）
│   ├── images/                 # 上传的图片
│   ├── css/custom.css          # 自定义样式
│   ├── js/
│   │   ├── comments.js         # 前台留言表单
│   │   ├── favorites.js        # 前台收藏
│   │   └── snow.js             # 雪花特效
│   └── favorites/index.md      # 收藏页面
├── themes/butterfly/           # 主题文件
├── scripts/
│   └── comment-api.js          # Hexo 本地开发时的留言 API 中间件
├── admin-panel/
│   ├── server.js               # 后台主程序 ⭐
│   ├── lib/content-filter.js   # 黄赌毒过滤规则
│   ├── public/
│   │   ├── index.html          # 后台单页应用
│   │   ├── js/admin.js         # 后台前端逻辑
│   │   └── css/admin.css       # 后台样式
│   └── data/
│       ├── auth.json           # 管理员密码（SHA256）
│       ├── deploy-token.txt    # GitHub 部署 Token（不提交 Git）
│       └── comments_*.json     # 留言数据（不提交 Git）
├── deploy/setup-server.sh      # 服务器一键安装脚本
└── docs/                       # 文档
```

### 4.4 server.js 模块划分

`admin-panel/server.js` 按功能分块，结构如下：

```
1. 静态资源服务
   express.static('public')  → 后台页面
   /images, /uploads         → 媒体文件

2. 认证 Auth
   POST /api/auth/login      → 登录，返回 token
   POST /api/auth/verify     → 验证 token
   POST /api/auth/change-password

3. 文章 Posts
   GET/POST/PUT/DELETE /api/posts
   → 读写 source/_posts/*.md（gray-matter 解析 frontmatter）
   → 保存非草稿时自动 runHexoDeploy()

4. 配置 Config
   GET/PUT /api/config/site      → _config.yml
   GET/PUT /api/config/theme     → _config.butterfly.yml
   GET/PUT /api/config/background

5. 主题文件 Theme Files
   GET/PUT /api/theme/file       → themes/butterfly/ 下文件

6. 媒体 Media
   GET /api/media
   POST /api/media/upload        → multer 上传 + 文件名过滤
   DELETE /api/media/:name

7. 留言 Comments（私密）
   GET /api/comments/:slug       → 公开 API 返回空数组
   POST /api/comments/:slug      → 保存留言
   GET /api/comments-all         → 管理员查看全部
   DELETE /api/comments/:slug/:id

8. Hexo 命令
   POST /api/hexo/generate
   POST /api/hexo/deploy
   POST /api/hexo/clean

9. 内容审核
   lib/content-filter.js         → 所有写入操作前调用
   POST /api/content-check       → 前端预检（可选）
```

### 4.5 核心流程：保存文章并发布

```
用户点「保存文章」
    │
    ▼
admin.js  POST /api/posts  { title, content, tags, ... }
    │
    ▼
server.js
    ├─ contentFilter.checkPost()  → 违规则 403
    ├─ matter.stringify() 写入 source/_posts/标题.md
    └─ 若非草稿：
         runHexoDeploy()
           ├─ hexo clean && hexo generate  → 生成 public/
           └─ hexo deploy                  → 推送到 xxf-f.github.io 仓库
                └─ 使用 deploy-token.txt 中的 GitHub Token
    │
    ▼
返回 { success, deployed, deployError }
    │
    ▼
admin.js  toast「文章已创建并发布到 GitHub」
```

### 4.6 留言跨域设计

前台在 `https://xxf-f.github.io`（HTTPS），API 在 `http://IP:4001`（HTTP）：

```javascript
// _config.butterfly.yml inject 注入
window.HEXO_MESSAGE_API = 'http://207.57.123.17:4001/api/comments';

// comments.js 读取
const API_BASE = window.HEXO_MESSAGE_API || '/api/comments';
```

服务器端开启 CORS：

```javascript
res.header('Access-Control-Allow-Origin', '*');
```

> 注意：HTTPS 页面请求 HTTP API 可能被浏览器拦截（混合内容）。若留言失败，需给服务器配置 HTTPS（如 Cloudflare Tunnel）。

### 4.7 主题配置可视化原理

后台不直接让用户编辑 YAML，而是：

```
加载：GET /api/config/theme → 解析 YAML → 填入表单字段
保存：表单字段 → buildThemeYaml() 拼出 YAML 字符串 → PUT /api/config/theme
```

`admin.js` 中 `buildThemeYaml()` 根据表单值生成 `_config.butterfly.yml` 全文，保证 `inject`、`disable_top_img` 等关键项不丢失（`ensureThemeEssentials()`）。

### 4.8 内容过滤模块

`lib/content-filter.js` 独立模块，规则分三类：色情、赌博、毒品。

```javascript
checkPost({ title, content, tags })   // 文章
checkComment({ nickname, content })   // 留言
checkRawContent(yamlString)           // 主题/站点配置
checkText(filename)                   // 上传文件名
```

英文关键词使用整词匹配（`\bmeth\b`），避免误伤 `method` 等正常配置项。

## 六、扩展开发建议

### 6.1 想加新后台功能

1. **后端**：在 `server.js` 增加路由，读写 `HEXO_ROOT` 下对应文件
2. **前端**：在 `index.html` 加页面区块，在 `admin.js` 加加载/保存函数
3. **如需前台展示**：在 `source/js/` 加脚本，在 `_config.butterfly.yml` 的 `inject` 引入

### 6.2 想加数据库

当前 JSON 文件存储适合低流量。若留言量大，可：

- 将 `readComments/writeComments` 改为 SQLite / MongoDB
- 保持 API 路径不变，前台无需修改

### 6.3 想加 HTTPS

推荐 Cloudflare Tunnel（免费）：

```bash
# 服务器安装 cloudflared，获得 https://xxx.trycloudflare.com
# 把 inject 里的 HEXO_MESSAGE_API 改成 https 地址
```

### 6.4 想换域名

1. 购买域名，DNS 指向服务器或 GitHub Pages
2. 修改 `_config.yml` 的 `url`
3. 更新 `inject` 中的 API 地址
4. 重新 `hexo deploy`

## 七、常见问题

### Q：后台发了文章，前台看不到？

**A：** 检查是否勾了草稿；是否部署成功（看保存后 toast）；手动点「部署上线」；确认 `deploy-token.txt` 有效。

### Q：主题保存报「毒品」误报？

**A：** 已修复 `method` 误匹配问题。若仍出现，检查主题配置是否含敏感词。

### Q：留言提交失败？

**A：** 可能是 HTTPS 页面无法请求 HTTP API。给服务器配 HTTPS，或本地测试时用 `hexo server`。

### Q：管理后台 404 / 打不开？

**A：** 确认 `admin-panel/public/index.html` 存在；PM2 状态 `pm2 status`；安全组放行 4001。

### Q：git pull 后后台异常？

**A：** `pm2 restart hexo-admin`
